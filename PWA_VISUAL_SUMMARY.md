## Behind The Scenes

### Service Worker Lifecycle

```
User visits app
     ↓
Service Worker installs
     ↓
Caches critical assets:
  ✓ index.html
  ✓ main.js
  ✓ styles.css
  ✓ manifest.json
     ↓
Service Worker activates
     ↓
App ready for offline use!

┌──────────────────────────────────┐
│  Next visit (Online):            │
│  1. Check cache first            │
│  2. Load instantly from cache    │
│  3. Fetch fresh data in bg       │
│  4. Update cache                 │
└──────────────────────────────────┘

┌──────────────────────────────────┐
│  Offline visit:                  │
│  1. Load everything from cache   │
│  2. Full app functionality       │
│  3. Queue changes locally        │
│  4. Sync when back online        │
└──────────────────────────────────┘
```

### Caching Strategy

```
Request Type          Strategy            Result
────────────────────────────────────────────────────
HTML/CSS/JS          Cache First         Instant load
Images               Cache First         Instant display
API Calls            Network First       Fresh data
  ↓ (if offline)     ↓ Cache Fallback    Cached data
Supabase             Network First       Real-time
  ↓ (if offline)     ↓ IndexedDB         Local storage
User Changes         Queue               Sync later
```
